# TourBots Website Guides Plan (Lean Version)

Aligned to the current website sections shown on the Guides page:
- Account Setup
- Tour Setup
- Chatbot Setup
- Share and Embed
- Analytics
- Billing

This is a reduced, practical list focused on core user outcomes.

## Account Setup

### 1) Create Your TourBots Account and Choose the Right Plan [DONE - seeded]
**What it should include:** Registration flow, Free vs Pro, and when to upgrade.  
**Key bits:** Start Free path, first login, plan limits, decision checklist.

### 2) First Login and Dashboard Overview [DONE - seeded]
**What it should include:** What users see first and what each main area does.  
**Key bits:** Dashboard stats, Tours, Chatbots, Settings, Help Centre, quick first actions.

## Tour Setup

### 1) Initial Tour Setup (From Zero to Live Tour) [DONE - seeded]
**What it should include:** Complete first-tour setup flow with minimum friction.  
**Key bits:** Add primary tour, connect Matterport model, save core settings, first validation checks.

### 2) How to Set Up Multiple Models in One Tour [DONE - seeded]
**What it should include:** Attach secondary models and ask AI to switch correctly between them.  
**Key bits:** Primary vs secondary model logic, setup workflow, common configuration mistakes.

### 3) How to Set Up Separate Tours (Multiple Locations) [DONE - seeded]
**What it should include:** Create and manage separate tours under one account, then configure a separate chatbot for each tour from the Chatbots page.  
**Key bits:** Location creation flow, selector behaviour, when to use separate tours vs multiple models.

### 4) How to Train AI on Navigation Points [DONE - seeded]
**What it should include:** Create positions and map AI behaviour to movement.  
**Key bits:** Save Position flow, naming standards, triggers/actions for navigation, test checklist.

### 5) Build a Guided Tour Menu [DONE - seeded]
**What it should include:** Create menu structure that drives user actions.  
**Key bits:** Tour menu blocks, position/model linking, recommended journey order.

### 6) Multiple Models vs Separate Tours: Which to Choose [DONE - seeded]
**What it should include:** A decision framework to help users choose the right setup for their use case.  
**Key bits:** When to use multiple models in one tour, when to create separate tours/locations, chatbot setup implications, analytics impact, and scaling guidance.
